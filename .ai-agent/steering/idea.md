# アイデア

現状のプロダクトに対する改善アイデア置き場。雑多にアイデアを書き、プランニングのためにどういうアイデアなのかを分類してより解像度を上げて整理しておく。

## 不具合

明らかに改善の必要がある・仕様に反している事象。

## 機能要望

検討の価値のある追加機能・既存機能の改善アイデア。

| 内容 | 温度感 |
|------|--------|
| タイトル自動生成。ID使うのをやめ、どういう画像なのか判別しやすくする | 中 |
| 閲覧・編集モード分離 | 中 |
| encodeURIComponent の使用をやめ、URLビルダを利用する | 中 |
| E2Eテスト追加 | 中 |
| URLからの画像クロール取り込み機能 | 中 |
| client / server での型の共通化 | 中 |
| 属性の発音埋め込みによる、検索性の向上 | 中 |
| 検索履歴からの検索補完 | 中 |
| 類似画像の説明を説明生成に反映 | 中 |
| 設定読み込みをグローバル変数でやるのをやめる | 低 |
| client パッケージを web-client パッケージに改名 | 低 |
| replan コマンド作成 | 低 |
| 複数サーバ対応 | 低 |
| 認証対応 | 低 |
| 多言語対応 | 低 |
| ヘルプページ作成。チュートリアル作成 | 低 |
| アップロードのリアルタイム進捗表示 | 低 |
| ジョブワーカーの導入 | 低 |
| URL 属性のリンク表示 | 低 |
