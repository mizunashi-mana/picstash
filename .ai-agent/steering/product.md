# Picstash プロダクト概要

## ミッション

**「インディーズメディア作品を楽しむ時間を作る」**

Pixiv、X、Spotify などに点在するインディーズ作品は、収集・管理・探索のコストが非常に高く、ファンが作品を「楽しむこと」そのものに没入できていない。picstash は、テクノロジーによって管理の事務的コストを消滅させ、没入の時間を最大化する。

---

## 最終ビジョン（3つの柱）

### 1. 媒体の網羅性（Omni-Platform Experience）

**「いつでも、どこでも、制約なく」**

- **デバイスの網羅:** PC、モバイル、Web、TV といった全ての媒体を網羅
- **メディア形式の網羅:** 画像、動画、音楽など、作品世界を構成する全ての形式を複合的に管理
- ユーザーの状況や気分に合わせて作品を鑑賞・管理できる環境
- オフラインとクラウドをシームレスに統合し、デバイスのストレージ制限から解放

### 2. 感性を理解したパートナー（Sentiment-Aware Partner）

**「探す・整理する手間を、感性で肩代わりする」**

- 単なる属性（タグ）管理ではなく、作品が持つ「空気感」「情緒」「文脈」を理解する AI
- ユーザーの潜在的な好みを察知し、能動的に探さずとも「今の自分」に最適な作品を提示
- 阿吽の呼吸を持つパートナー

### 3. 没入感をもたらす創造性（Immersive Creativity）

**「作品世界を拡張し、没入の深さを変える」**

- AI による二次創作生成、キャラクターデザイン、音声生成などの創作的補助機能
- 既存の作品を起点に「自分だけの物語」を拡張・深化
- 私的利用の範囲での圧倒的な没入体験

- 同人作家の創作物
- 家族の写真
- 知人の創作物
- 大手プラットフォームで配信されていない、**個人的な価値を持つメディア**を愛する人々

## 市場課題

> **自分の感覚を言葉にして入力しなければ、欲しいコンテンツに辿り着けない**

この課題は様々な背景から生じている：

| 背景 | ビジョンによる解決 |
|------|-------------------|
| コレクションが複数の場所に散らばり、統合されていない | **媒体の網羅性:** 制約なくコレクションを構築・管理できる基盤 |
| デバイスやメディア形式の制約で、触れたいコンテンツに触れられない | **媒体の網羅性:** いつでもどこでも、あらゆる形式で触れられる |
| 整理に時間がかかりすぎる | **感性パートナー:** 能動的な整理なしに、AIが自動で理解・整理 |
| 既存ツールは「言語化されたもの」での検索を前提としている | **感性パートナー:** 言語化なしに感覚で探索・提案 |
| 欲しいコンテンツがそもそも存在しない場合もある | **没入感をもたらす創造性:** 存在しなければ生成する |

→ **3つの柱が揃って初めて、この課題が解決される**

**「フラっと使い始め、いつの間にか手放せなくなる」**

### 利益モデル

| 収益源 | 位置づけ | 内容 |
|--------|----------|------|
| サブスクリプション | 基盤収益 | AI アップデート、基本容量、プラットフォーム追従への対価 |
| ストレージ課金 | **主要利益源** | コレクション拡大に伴う従量課金。AI インデックス付きの差別化 |

### なぜストレージが主要利益源になるか

- ユーザーが「手放せなくなる」ほど、コレクションは増え続ける
- コレクションが増えるほど、AI の感性理解は深まる（＝離脱障壁が高まる）
- 単なる「容量」ではなく「中身を理解している倉庫」として差別化
- コレクションの成長 = 収益の成長 = LTV の最大化

## プロダクト戦略

### 戦略ロードマップ

```
Phase 1                 Phase 2                 Phase 3                 Phase 4
デスクトップアプリ        ブラウザプラグイン        クラウド対応             モバイル対応
────────────────────────────────────────────────────────────────────────────────────

[感性マップ体験]    →   [外部連携・取り込み]  →  [マルチデバイス同期]  →  [いつでもどこでも]

・ローカル AI           ・Pixiv / X 連携         ・クラウドストレージ      ・iOS / Android
・類似画像探索          ・1クリック取り込み       ・デバイス間同期          ・モバイル最適化 UI
・芋づる式発見          ・メタデータ自動取得      ・バックアップ           ・オフライン対応

────────────────────────────────────────────────────────────────────────────────────
キャッシュフロー: 軽     キャッシュフロー: 軽      キャッシュフロー: 重      開発コスト: 重
                                                     ↑
                                              モバイルはここに依存
```

**ポテンシャル順位:** モバイル（最大） > クラウド > プラグイン > デスクトップ

**実行順位:** デスクトップ → プラグイン → クラウド → モバイル（依存関係による）

### 戦略の背景

#### なぜデスクトップアプリから始めるか

| 観点 | 理由 |
|------|------|
| **市場適合性** | 「AI × 受動的」の象限に競合がおらず、差別化が明確 |
| **キラー機能** | ローカル LLM での感性マップ体験は他にない価値提案 |
| **キャッシュフロー** | クラウド不要でインフラコストが軽い |
| **技術リスク** | ローカル完結で外部依存が少なく、開発・運用がシンプル |

#### なぜブラウザプラグインが次か

| 観点 | 理由 |
|------|------|
| **ユーザー価値** | 感性マップの価値を実感した後、「もっと入れたい」欲求に応える |
| **キャッシュフロー** | デスクトップアプリとの連携で完結、クラウド不要 |
| **テーマ** | 外部連携（Pixiv、X）とコレクションへの取り込み |

#### なぜクラウド対応が Phase 3 か

| 観点 | 理由 |
|------|------|
| **必要性** | モバイル対応の前提条件、マルチデバイス体験の基盤 |
| **キャッシュフロー** | インフラコストが発生、収益基盤の確立が先に必要 |
| **テーマ** | ストレージ課金モデルの本格稼働 |

#### なぜモバイルは最後か

| 観点 | 理由 |
|------|------|
| **ポテンシャル** | **最も大きな市場**、「いつでもどこでも」の体験に必須 |
| **依存関係** | クラウド対応が必須であり、Phase 3 完了後でないと着手不可 |
| **開発コスト** | iOS / Android 両対応、モバイル最適化 UI が必要で重い |
| **順序** | ポテンシャルは最大だが、依存関係により最後になる |

### Phase 1：デスクトップアプリ × 感性マップ体験

最終ビジョンに向けた第一歩として、PC 媒体でのリッチな体験と深いユーザー理解の構築に集中する。

**テーマ:** ローカル AI による感性マップ体験

#### ターゲットペルソナ（Phase 1）

| 項目 | 内容 |
|------|------|
| 属性 | インディーズ作品を愛する鑑賞者/コレクター（創作はしない） |
| コレクション規模 | 数千〜数万枚、数年分の蓄積 |
| 保存状況 | ローカルフォルダ、クラウド、複数の場所に散在 |
| 現状の行動 | ファイル名検索 → 見つからない → 諦めて新しい作品を見に行く |
| 本当の欲求 | 「あの雰囲気」「前に見たあれ」で過去の宝に再会したい |
| 探し方 | 言葉にしにくい感覚、記憶の断片から |

#### ユーザーストーリージャーニー

**Phase 0: 認知**

| 状況 | 心理 |
|------|------|
| 友人の口コミ or「画像 整理 AI」で検索 | 「散らばった画像、なんとかしたいな」 |
| picstash を発見 | 「感覚で探せる？試してみるか」 |

**Phase 1: 試用（最重要）**

| 行動 | 心理 |
|------|------|
| 数十枚を取り込む | 「どんなもんかな」 |
| 即座に一部が整理される | 「おっ、速い」 |
| 徐々に発見が増えていく | 「まだ何か出てくる」 |
| 背景情報（作者名など）が自動付与 | 「わかってる」 |

**離脱ポイントと対策:**

| リスク | 対策 |
|--------|------|
| 取り込みに時間がかかる | 即時に一部表示 + 残りはインクリメンタルに反映 |
| 整理結果が期待と違う | 複数ソースから情報取得、コレクション横断学習で精度向上 |

**Phase 2: 価値実感**

| 行動 | 心理 |
|------|------|
| 「あの雰囲気」で探してみる | 「見つかった！これこれ」 |
| 1枚選ぶと芋づる式に関連が出る | 「こんなのもあったんだ」 |
| 似た雰囲気の画像がグループ化されている | 「自分ってこういうの好きだったんだ」 |

→ **「再発見の快感」** を得る

**Phase 3: 拡張**

| 行動 | 心理 |
|------|------|
| もっと入れたくなる | 「他のフォルダも入れてみよう」 |
| 散らばったコレクションを統合 | 「全部ここにまとめたい」 |

**Phase 4: 習慣化**

| 行動 | 心理 |
|------|------|
| 「見たい気分」の時に開く | 「今日はどんな発見があるかな」 |
| 開くたびに静かに進化している | 「また何か見つかってる」 |
| セッション体験（察知→編合→帰着） | 「いい時間だった」 |

**Phase 5: 手放せない**

| 状態 | 心理 |
|------|------|
| 自分の感性を一番理解している存在 | 「ここにしかない繋がりがある」 |
| 他ツールでは再現できない蓄積 | 「もう戻れない」 |

#### コア体験：感性のネットワークを通じた「自己再発見」

単なる整理ツールではなく、**「自分の感性を映し出す鏡」** としての価値を提供する。

**1. 摩擦ゼロの収集（Passive Collection）**

- ドラッグ＆ドロップ、フォルダ指定で一括取り込み
- 将来的にはブラウザプラグインで 1 クリック取り込み
- ユーザーは整理を意識せず、ただ「いいな」を投下するだけ

**2. 徐々に育つ整理（Incremental Organization）**

- 取り込み直後：即座にサムネイル表示 + 一部の整理結果
- バックグラウンド：残りの画像を順次解析・反映
- 翌日以降：開くたびに静かに発見が増えている（通知はしない）

**進捗の体感:**

機械的な進捗バー（「解析中 42/100」）ではなく、**体験として進捗を感じる**設計とする。

| 体感の例 | 内容 |
|---------|------|
| おすすめが徐々に変わる | 最初は汎用的 → 使うほど「自分向け」に |
| グループ化が洗練される | 最初は大雑把 → 徐々に細かい繋がりが見える |
| 芋づる式の精度が上がる | 最初は遠い関連 → 徐々に「まさにこれ」が出る |
| 新しい繋がりの発見 | 「この2枚、実は似てるかも」が増えていく |

→ ユーザーは「数字」ではなく「体験の質の変化」で進捗を感じる

**3. 多層的な情報取得**

| ソース | 取得情報 |
|--------|----------|
| EXIF / メタデータ | 作成日時など |
| AI 画像解析 | キャラクター、構図、色調、雰囲気 |
| 外部サービス連携 | Pixiv、X から作者名・タイトル |
| コレクション横断学習 | 「この絵師の他の作品と似ている」など |

**4. 感覚での探索**

- 似た雰囲気の画像がグループ化
- 1枚選ぶと芋づる式に関連画像が出てくる
- 「あの雰囲気」「前に見たあれ」で探せる

**5. セッション独立型の鑑賞体験（将来）**

- **起動（Fresh Start）:** 毎回フレッシュな状態で迎え、過去の文脈を強制しない
- **察知（Sensing）:** 最初に手に取った1枚から「今の空気感」を読み取る
- **編合（Choreography）:** 今の気分に呼応する作品を繋ぎ合わせ、自然なストーリーとして上映
- **帰着（Landing）:** 15〜30分の旅の終わりに、心地よい余韻と共に現実へ送り出す

### 技術方針

#### アーキテクチャ

| 項目 | 方針 |
|------|------|
| 実行環境 | 完全ローカル（プライバシー重視、オフライン対応） |
| データベース | SQLite（軽量、ローカル完結） |
| AI 処理 | バックグラウンド非同期、インクリメンタル反映 |

#### AI モデル構成

| 用途 | モデル / 技術 |
|------|-------------|
| 画像埋め込み・類似検索 | CLIP |
| 説明生成 | ローカル LLM |
| 認識 | 顔認識 / キャラクター認識 |

#### 設計原則

- **即時レスポンス:** 取り込み直後に一部結果を表示、残りはバックグラウンドで
- **静かな進化:** 通知せず、次回起動時に自然と発見が増えている
- **プライバシーファースト:** データは完全ローカル、外部送信なし

### 重要な課題・リスク

#### 1. 「1%の的外れ」問題

**課題:**
AI が「わかってる」とユーザーに感じさせるには、90%の正解ではなく、**1%の「的外れ」を排除する**方が重要。感性を謳う以上、一箇所でも「センスのない推奨」が出た瞬間に、ユーザーの没入感は冷めてしまう。

**解決の方向性:**

| アプローチ | 内容 |
|-----------|------|
| 控えめな提案 | 「押し付け」ではなく「発見の種」として提示 |
| 信頼度の活用 | 確信度が低い推奨は表示しない、または控えめに表示 |
| フィードバック学習 | 的外れをユーザーが修正できる仕組み、学習への反映 |
| 徐々に精度向上 | 最初は慎重に、コレクション横断学習で精度を上げていく |

**状態:** 解決策の検証が必要。Phase 1 での重要な検証項目。

#### 2. 著作権と AI 生成

**課題:**
「没入感をもたらす創造性」の柱における二次創作支援は、インディーズ作品を扱う性質上、**権利関係が非常にデリケート**。「私的利用の範囲」であることを技術的・規約的にどう担保するかが重要。

**解決の方向性:**

| 観点 | アプローチ |
|------|-----------|
