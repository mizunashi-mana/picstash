# plan.md の市場分析を受けての見直し

## 目的・ゴール

市場分析（market.md）の結果を受けて、plan.md のロードマップを見直し、より現実的で将来性のある計画に更新する。

## 背景

market.md の分析により、以下の重要な示唆が得られた:

1. **最優先ターゲットはイラスト管理層** - Eagle/Hydrus の弱点を補完できる
2. **デスクトップアプリ対応を短期で推奨** - Eagle からの移行を狙う
3. **同人作品管理向け機能の需要** - 漫画ビューア、シリーズ管理（ただし後回し）
4. **モバイルアプリ・クラウド同期の長期需要**

## 変更方針

### 新しい Phase 構成

| Phase | テーマ | 詳細度 | 備考 |
|-------|--------|--------|------|
| 1 | デスクトップアプリ化 | 詳細 | 短期・最優先 |
| 2 | 動画対応 | 詳細 | 短期・デスクトップアプリ後 |
| 3 | おすすめ機能拡充 | 詳細 | 一部実装済み |
| 4 | メディア取り込み強化 | 詳細 | ブラウザ拡張・D&D |
| 5 | 学習データ活用 | 詳細 | 一部実装済み |
| 6 | クラウド化 | 方向性のみ | 長期・セルフホスト拡張 |
| 7 | モバイルアプリ | 詳細 | 長期・クラウド前提 |
| 8 | 販促 | 方向性のみ | 長期 |

### Phase 4: メディア取り込み強化

- ブラウザ拡張で表示中のページの画像を選択して取り込み
- ページ内の全画像を一括でインポート
- 出典 URL・ページタイトル・作者情報の自動取得
- デスクトップアプリへの直接ドラッグ＆ドロップ

### Phase 6: クラウド化（方向性）

- サーバの冗長性
- 認証・認可（マルチユーザー対応）
- 管理画面の作成
- 他サービスとの連携
- モバイルアプリの前提基盤

### Phase 7: モバイルアプリ

- クラウド基盤を前提としたモバイル対応
- レスポンシブ UI → PWA → ネイティブアプリ

### Phase 8: 販促（方向性）

- ヘルプページ・チュートリアル拡充
- 公式 Web サイト公開
- アプリを全面的にプロモーション

### 将来の展望セクション（新規追加）

- SaaS 化: 専用クラウドストレージサービスとしての提供
- Phase 7 終了後にロードマップを再策定予定

### 除外した項目

- コミュニティ機能（共有・コラボレーション）は目指さない
- シリーズ管理・漫画ビューアは Phase として入れず、将来的な検討事項に留める
