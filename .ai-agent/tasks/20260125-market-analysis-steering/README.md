# 市場分析と steering ドキュメント追加

## 目的・ゴール

Picstash の市場ポジショニングを明確にし、steering ドキュメントに市場分析結果を追加する。
この分析結果は、後続タスクで plan.md の実装計画を見直す際の判断材料となる。

## 実装方針

### 1. ターゲット市場の調査（市場セグメント別）

以下の切り口で市場を分析し、それぞれのニーズ・課題・競合を整理する：

| セグメント | 想定ユーザー | 主な用途 |
|-----------|-------------|---------|
| 写真管理 | 一般ユーザー、カメラ愛好家 | 撮影した写真の整理・検索・共有 |
| イラスト管理 | イラストコレクター、絵師 | 収集したイラスト・自作品の整理 |
| 同人作品管理 | 同人誌・二次創作の収集者 | 作品のカタログ化、シリーズ管理 |
| 参考資料管理 | デザイナー、クリエイター | インスピレーション素材の収集・分類 |
| デジタルアート収集 | NFT・デジタルアート愛好家 | 購入作品の管理・証明 |

各セグメントで調査する項目：
- **市場規模・成長性**: どのくらいのユーザーがいるか
- **ユーザーペルソナ**: 典型的なユーザー像
- **主要なニーズ**: 何を解決したいか
- **現状の課題**: 既存ツールで解決できていないこと
- **支払い意欲**: 有料ツールへの抵抗感

### 2. 競合分析

#### 調査対象ツール

| カテゴリ | ツール例 |
|---------|---------|
| 汎用画像管理 | Eagle, Hydrus Network, digiKam, XnView |
| 写真管理 | Adobe Lightroom, Google Photos, Apple Photos |
| イラスト特化 | PureRef, Kuadro, Pinterest |
| DAM (企業向け) | Adobe Experience Manager, Bynder |
| 自作/OSS | Immich, PhotoPrism, Lychee |

#### 分析項目

**A. 機能比較**
| 機能 | 調査ポイント |
|------|-------------|
| インポート | 対応形式、一括取込、重複検出 |
| 整理・分類 | タグ、フォルダ、コレクション、スマートアルバム |
| 検索 | キーワード、フィルタ、類似検索 |
| AI機能 | 自動タグ、顔認識、説明生成 |
| 閲覧 | ギャラリー表示、詳細表示、スライドショー |
| 連携 | クラウド同期、外部サービス連携 |
| デプロイ | ローカル/クラウド/セルフホスト |

**B. 既存ユーザーの不満点**

以下のソースから不満・要望を収集：
- **レビューサイト**: G2, Capterra, Product Hunt
- **フォーラム/Reddit**: r/DataHoarder, r/selfhosted, 各ツールのサブレディット
- **SNS**: Twitter/X での言及
- **GitHub Issues**: OSS ツールの feature request, bug report

収集する不満の観点：
| 観点 | 例 |
|------|-----|
| パフォーマンス | 大量画像で重い、検索が遅い |
| UI/UX | 使いにくい、学習コストが高い |
| 機能不足 | 〇〇ができない、△△に対応していない |
| 価格 | 高すぎる、サブスクが嫌 |
| プライバシー | クラウド強制、データ収集が不安 |
| 安定性 | バグが多い、開発が停滞 |
| プラットフォーム | 特定OSのみ、モバイル非対応 |

**C. ビジネスモデル比較**
| ツール | 価格モデル | 価格帯 |
|--------|-----------|--------|
| Eagle | 買い切り | $29.95 |
| Lightroom | サブスク | $9.99/月〜 |
| Hydrus | 無料/OSS | - |
| etc. | | |

### 3. Picstash の強み・差別化分析

現在の機能と市場ニーズを照らし合わせ：
- **独自の強み**: 他ツールにない価値
- **改善の余地**: 競合に劣る点
- **差別化の方向性**: どのセグメントに注力すべきか
