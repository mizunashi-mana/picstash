# start-new-project コマンドの実装

## 目的・ゴール

`start-new-task` より長期的な目標を管理するための `start-new-project` コマンドを作成する。

## 実装方針

### プロジェクトディレクトリ構造

```
.ai-agent/projects/YYYYMMDD-project-name/
└── README.md    # 目標・スコープ・タスク分解・進捗を1ファイルに
```

### コマンドの処理フロー

1. プロジェクト名の決定（kebab-case）
2. プロジェクトディレクトリ作成
3. 関連ドキュメント確認（plan.md の該当箇所など）
4. 目標・スコープの明確化
5. タスク分解と依存関係の整理
6. 依存関係の妥当性チェック
7. README.md 作成
8. ユーザー確認
9. （任意）最初のタスクを `start-new-task` で開始

### 設計方針

- **1ファイル完結**: 計画・進捗・メモを README.md に集約
- **柔軟な調整**: タスク分解は随時追加・変更・並び替え可能
- **明確な優先順位**: 何から取り組むかが一目でわかる
- **依存関係の管理**: タスク間の依存を明記し、着手可能なタスクを明確に
- **軽量な進捗管理**: 日付と出来事を箇条書きで記録
